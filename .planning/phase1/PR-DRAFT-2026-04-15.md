### Bug fixes (from harness-reviewer audit)

- `7a169fb` тАФ **B1 (BLOCKING)** ChatPanel + ImportDialog never passed `sessionId` on `/api/chat`; server was silently skipping the entire brainstorm-state persistence layer. Fixed: every request carries `sessionId`.
- `7a169fb` тАФ **B2 (BLOCKING, stored XSS)** OptionCards rendered LLM-authored option text through `dangerouslySetInnerHTML` + regex markdown. Replaced with a safe React-element renderer.
- `11d8df1` тАФ **Brainstorm B1 (BLOCKING)** `DECISIONS_RE` non-greedy regex truncated nested `tech_preferences` JSON at the first `}`, silently dropping every patch with nested keys. Now captures through `-->`.
- `11d8df1` тАФ **Brainstorm B2 (BLOCKING)** `tech_preferences` was shallow-merged, clobbering prior turns. Now deep-merges per key while domain/scale stay last-write-wins.
- `11d8df1` тАФ **Brainstorm B3 (BLOCKING)** Concurrent `POST /api/chat` with the same `sessionId` raced on read-modify-write. Fixed with `updateBrainstormState` (per-session promise chain + rebase-on-current) + 15-case integration test at `tests/integration/brainstorm-e2e.test.ts`. Cross-process mutex still open (YAGNI until multi-worker deploy).
- `53a9f0c` тАФ **B3 (IDB)** `idbSaveSessions` had no `tx.onabort`; tab kill between `store.clear()` and `oncomplete` wiped mirror silently. Added.
- `53a9f0c` тАФ **B4** `flushSave` dropped the `_doSave` promise on `beforeunload`. Returns the promise now.
- `92bd673` тАФ **Brainstorm pre-restart #1** dedup key was `service+envVar`, collided across distinct dep kinds (e.g. oauth_app vs api_key sharing a service name). Rekeyed to `service+type+envVar` with a `service+envVar` fallback lookup for remove ops that omit type.
- `92bd673` тАФ **Brainstorm pre-restart #3** `PROGRESS_RE` used `[^>]*?`; aligned with DECISIONS_RE fix (`[\s\S]*?` up to `-->`).
- `92bd673` тАФ **Brainstorm pre-restart #4** Skill spec emits `action` (oauth_app) and `note` (compliance) fields, but zod schemas only allowed `notes`. Added `action` + accepted singular `note` (folded into `notes`) in reduce. OAuth + compliance deps keep their human-facing descriptions.
- `028ff98` тАФ **S4** `roundCount` could regress on LLM hallucination emitting lower `round=N`. Guarded with `Math.max(prev, patch ?? 0)`.
- `028ff98` тАФ **N defense-in-depth** `ChatMarkdown` explicit `urlTransform` whitelist: `https?` / `mailto` / in-page anchors / site-relative only; drops `xmpp` / `ircs` the app doesn't need.
