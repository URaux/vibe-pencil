构建流程 Build Flow

用户点击 "Build All" 后的完整执行链路 — 计划预览、波次调度、SSE 流式进度、Post-build hooks 到 BuildSummary 反馈。

01 👤 用户 User 点击 "Build All"
画布工具栏触发全量构建。 User clicks Build All on canvas toolbar.
plan preview
02 🎨 画布 Canvas Build Plan Preview — 构建计划预览
显示每个节点匹配的 Skill 和可编辑的 workDir，用户可在执行前调整。

API Server fastapi-skill ./backend editable
Database postgres-skill ./db editable

03 🔀 TopoSort 拓扑排序 → Waves
Kahn 算法计算依赖图，输出有序执行批次。
Wave 0 无依赖节点   Wave 1 依赖 Wave 0   Wave N 串行执行

05 🔴 SSE 流 Stream 实时进度推送
AgentRunner 将 AI CLI 输出实时转发到 Canvas 进度面板。

post-build hooks
06 ⚙️ Post-build Hooks 构建后自动执行 Hooks
每个节点构建完成后触发 Skill 定义的后置命令。
lint test format custom cmd
失败时节点标记为 hook-failed，可重试
extract summary
07 🤖 AgentRunner BuildSummary 提取
从 AI CLI 输出中提取结构化构建摘要。
files created commands run errors / warnings
inject to chat
08 🎨 Canvas Agent 构建事件自动插入对话
BuildSummary 自动注入到 Chat 上下文（L3 Canvas State）。
Canvas Agent 可读取构建结果，用户可在 Chat 中直接追问构建细节。
build events → L3 Chat aware of build state
